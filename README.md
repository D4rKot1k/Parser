# Parser
Разработайте консольный скрипт-парсер, который будет получать данные с любого открытого публичного API 
(например, [https://jsonplaceholder.typicode.com/posts](https://jsonplaceholder.typicode.com/posts)) или парсить заголовки новостей 
с новостного сайта (например, используя `requests` и `beautifulsoup4`).
Технические требования:
1. Основы Python:
    - Скрипт должен использовать как минимум 2 различных структуры данных из модуля `collections` (например, `defaultdict` для группировки данных или `Counter` для подсчета).
    - Использовать как минимум один декоратор (например, для логирования времени выполнения функции или обработки ошибок).
    - Код должен быть оформлен в виде модуля. Конфигурационные данные (URL, путь для сохранения) должны выноситься в отдельный файл `config.py`.
2. ООП:
    - Реализовать минимум два класса. Например:
        - Parser: отвечает за загрузку и первичную обработку данных.
        - DataManager: отвечает за сохранение и обработку данных (например, группировку, фильтрацию). 
    - Продемонстрировать один из принципов SOLID на практике (лучше всего - Принцип единственной ответственности, разделив логику парсинга и управления данными).
3. Git:
    - Создайте публичный репозиторий на GitHub для вашего проекта.
    - Проект должен иметь как минимум 3 осмысленных коммита (например: "Добавлен базовый класс Parser", "Добавлен декоратор для логирования", 
                                                             "Рефакторинг: разделение логики по классам").
    - Создайте файл `.gitignore`, исключающий служебные файлы (например, `pycache/, .env`).
    - _(Опционально, но желательно)_ Настройте `pre-commit` хуки для автоматической проверки кодстайла с помощью `flake8`.
